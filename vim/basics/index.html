According to vim.org

Vim is a highly configurable text editor built to make creating and changing any kind of text very efficient. It is included as “vi” with most UNIX systems and with Apple OS X.

Vim is rock stable and is continuously being developed to become even better. Among its features are:

  • persistent, multi-level undo tree
  • extensive plugin system
  • support for hundreds of programming languages and file formats
  • powerful search and replace
  • integrates with many tools

Open vim. Type :help. Press <ENTER>

What vim is excellent at? Navigate. Make changes. Repeat.

Vim achieves all of its features because it supports different modes.

NORMAL

By default, Vim starts in normal mode.

In normal or command mode you can swiftly navigate around your file, manipulate text and launch commands.

Wtf you mean by launching a command?

Command Description
:s/old/new/g change ‘new’ to ‘old’ where g is globally
:!ls run a shell command
:read ls read a shell commands output
:Explore open vim’s builtin file manager
:Terminal open vim’s builtin terminal

The black-hole of the key bindings.

In NORMAL mode, literally every key on your keyboard does something. There is a lot of vim cheatsheets out there so i won’t list all of the keys.

INSERT

This mode will be familiar, because INSERT mode’s behaviour is similar to the normal behaviour of other text editors.

Once you are in INSERT mode, typing inserts characters just like a regular text editor.

VISUAL

This mode is used to make selections of text, similar to how clicking and dragging with a mouse behaves. Selecting text allows commands to apply only to the selection, such as copying, replacing and so on.
